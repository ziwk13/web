CDN

1. Contents Delivery Network
2. 네트워크상에 존재하는 콘텐츠를 사용하는 방식입니다.
3. 전 세계 여러 지역에 분산된 서버(엣지 서버, POP 등)를 통해 웹 콘텐츠(HTML, 이미지, 동영상 등)를 사용자에게 빠르고 효율적으로 전달하는 네트워크 시스템입니다. 
4. CDN의 핵심 목적은 웹사이트의 로딩 속도를 높이고, 서버의 부하를 분산시키며, 대역폭 사용을 최적화하고, 보안과 안정성을 강화하는 데 있습니다.
5. 대규모 웹사이트, 동영상 스트리밍 서비스(유튜브, 넷플릭스 등), 이미지 제공 사이트, 게임 파일 다운로드 등 빠른 콘텐츠 전송이 중요한 서비스에서 널리 사용됩니다.
6. CDN의 주요 구성 요소
  1) 엣지 서버(Edge Server)
    전 세계 여러 지역에 분산되어 사용자와 가까운 위치에서 콘텐츠를 제공하는 서버
  2) 원본 서버(Origin Server)
    실제 콘텐츠가 저장된 서버
  3) 캐시(Cache)
    자주 요청되는 콘텐츠를 임시로 저장하는 저장소
  4) 라우팅 시스템
    사용자의 요청을 최적의 엣지 서버로 안내하는 역할
7. CDN의 동작 원리
  1) 사용자가 웹사이트에 접속하면, CDN은 사용자의 위치와 가장 가까운 엣지 서버에서 콘텐츠를 제공합니다.
    이 서버들은 미리 콘텐츠를 캐싱(저장)해 두거나, 최초 요청 시 원본 서버(Origin Server)에서 받아와 저장한 뒤 사용자에게 전달합니다.
  2) 만약 엣지 서버에 해당 콘텐츠가 없다면, 원본 서버에서 받아와 제공하고, 이후 요청부터는 캐싱된 콘텐츠를 빠르게 제공합니다.
8. CDN 등록 방식
  1) CSS
    @import url("주소")
  2) HTML
    <link href="주소">
9. 주요 무료 CDN 제공업체
  1) jsDelivr
    오픈소스 프로젝트와 퍼블릭 GitHub 저장소 파일을 무료로 배포할 수 있는 CDN입니다. 빠른 속도와 쉬운 사용법이 장점입니다.
  2) Cloudflare
    글로벌 네트워크를 기반으로 빠르고 안정적인 CDN 서비스를 제공합니다. 무료 플랜도 제공하며, 웹사이트 최적화 및 보안 기능도 포함되어 있습니다.
  3) 카카오 CDN
    티스토리 등 카카오 계열 서비스에서 제공하는 CDN입니다. 일부 서비스에 한정되어 있지만, 국내에서 빠른 속도를 제공합니다.
